# Автоматизация формы sixmo.ru

Этот skill автоматически проходит многошаговую тестовую форму на https://sixmo.ru/

## Что делает

1. Открывает сайт sixmo.ru в браузере (Playwright + Chromium)
2. Обходит детекцию автоматизации (Playwright bindings, webdriver flag)
3. Имитирует реальные взаимодействия (mouse movement, keyboard timing, focus sequence)
4. Нажимает "Начать задание"
5. **Семантически** заполняет Этап 1 — определяет поля по label-тексту вопроса
6. **Семантически** заполняет Этап 2 — загрузка файла + текст + select
7. Отправляет форму и возвращает идентификатор результата

## Как использовать

### Базовый запуск (все ответы из placeholder'ов)

```bash
node sixmo-form.js
```

### С указанием ответов по тексту вопросов

Ответы привязываются к **тексту вопроса** (label), не к порядку полей:

```bash
node sixmo-form.js --data='{"answers":{"сову":"Букля","факультет":"Гриффиндор","школа":"Хогвартс","платформ":"Платформа 9 3/4","квиддич":"Снитч"},"file_path":"./my_file.txt"}'
```

Ключи в `answers` — это **подстроки из текста вопроса**. Например:
- `"сову"` матчит вопрос "Как звали **сову** Гарри Поттера?"
- `"факультет"` матчит "На какой **факультет** распределили..."
- `"квиддич"` матчит "Какой из этих предметов связан с **квиддич**ем?"

### Параметры

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `answers` | Объект `{"подстрока вопроса": "ответ"}` | Из placeholder / первый вариант |
| `file_path` | Путь к файлу для загрузки (.txt/.md/.json до 50 КБ) | Временный файл "Expelliarmus" |
| `headless` | Запуск без UI | `false` |

### Результат

```json
{
  "success": true,
  "identifier": "24E0AF1831CB",
  "flowId": "abc123...",
  "url": "https://sixmo.ru/#/flow/.../result",
  "screenshot": "sixmo_result.png"
}
```

## Особенности реализации

- **Семантическая привязка**: поля идентифицируются по тексту label (вопроса), а не по порядку или DOM-структуре
- **Обход детекции**: удаляет `__playwright__binding__`, `__pwInitScripts`, скрывает `navigator.webdriver`
- **Имитация взаимодействий**: mouse move (с промежуточными шагами), scroll, keyboard timing с вариативными задержками, focus sequence
- **Адаптивность**: поля могут быть в любом порядке — скрипт находит каждое по типу и label
